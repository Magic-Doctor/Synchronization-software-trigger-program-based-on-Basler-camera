​其中涉及到pylon的API,可去官网下载SDK。MultiCamAcqu_Basler.py是主程序。

basler官方的软件pylon Viewer给出了以“Action Commands”的方式来同步多相机单次采集的方案，但是这个方案只能单次同步采集，并且每个相机还要手动保存图像，过于繁琐，对于需要一定帧率自由连续同步采集的场景并不好用。因此对此进行了二次开发。主要实现的功能就是多相机连续同步采集，当然也具备基本的单次同步采集能力。

1. 相机理论连接数量没有上限，多相机连接时需要借助交换机。

2.相机连接上后会自动识别相机的id，每个相机在采集前必须要设置图片存储位置

3. 曝光时间可通过滑动条或者数字调节，相机的帧率超过相机上限改大了也没用

4. 图像尺寸改的是视场显示的区域，而不是像素分辨率

5. 程序可以实现自由连续采集，同步单次采集和同步连续采集三个功能。

自由连续采集：每个相机自由连续采集，不保持图像

同步单次采集：每个相机同步采集单张图片并保存

同步连续采集：每个相机连续同步采集多张图片并自动保存

6. 尝试了官方给的PTP协议来调控同步时差，不过没做好，功能可以忽略，没用

注意细节：

1. 自由连采时可以不用设置图片保存地址，如果设置了，在自由连采时会持续保存图片在电脑上。

2. 有时候同步连采会因为数据传输不稳定，出现图像丢失，可能会导致某个相机的某张图像漏掉，只要保证每个相机的图像编号是对上的那么这些图像就是同步性对应的（或者说时间戳是同步的）。比如相机1存储是 1 2 3 5 6，相机2存储是 12 3 6，两个相机的图像1,2,3, 6还是同步的。通过图像编号判断能避免相机因传输不稳定导致的图像采集对应性问题。

3. 程序核心是软件触发，因此传输线的带宽限制、代码执行效率，硬件条件等因素都可能会影响同步性，因此该程序比较适合低速或者静止状态的多视角测量。

用两个相机测试了下程序的同步性，硬件是千兆网线以及常规笔记本，采用手机上的毫秒表作为标度，结果显示程序实现的同步效果还是有效的。多次测试下来的结果，同步误差大概在20ms-50ms左右。
